## Milestone Summary

**Timeline:** 2026-02-23 → 2026-03-15 (21 days)

**Key Decisions:**
- Credit-based pricing: Lower commitment for users, easier to test profitability
- 3-tier analysis strategy: Balance cost vs accuracy, let users choose (Basic/Premium/Forensic)
- Serverless deployment (Vercel): Zero infra management, scales automatically
- Async job processing (BullMQ): Long-running AI jobs need queue, not HTTP
- PDF export for results: Users want portable, shareable reports
- Forensic tier buffer allocation: 5k token buffer for Forensic (vs 2k for Basic/Premium)
- PDF generation: Server-side with pdfkit, cached in Supabase Storage
- History filters: Native date inputs, instant application (no "Apply" button)
- Demo rate limiting: 5 requests/day per IP (user feedback adjustment)

**Issues Resolved:**
- Fixed new user credits issue (3 → 0) enabling email verification to award 10 credits
- Fixed demo rate limiting (10/hour → 5/day per IP) per user feedback
- Fixed tooltips triggering together and tier cards expanding together
- Fixed upload payload limit for 10MB files
- Fixed analysis status realtime updates with polling fallback
- Fixed PDF emoji garbled, footer missing, date filter issues

**Technical Debt Incurred:**
- BullMQ workers must run on Railway/Render, not Vercel serverless (timeout risk)
- Token usage baseline unknown — need to log first 100 analyses
- RLS policies need testing before production user data storage

**Requirements Coverage:**
- v1 requirements: 20 total
- Completed in v1.0: 17/20 (85%)
- Remaining: ADMIN-01/02/03/04 (Phase 6), DEPLOY-01 (Phase 7)
