  <name>Task 1: Create cost analysis API endpoint with margin calculations</name>
  <files>server/api/admin/costs.get.ts</files>
  <action>
    Create new endpoint `/api/admin/costs` that returns cost analysis per tier with profit margins.

    Query parameters:
    - `range`: '7d' | '30d' | '90d' | 'all' | 'custom'
    - `from`: optional ISO date for custom range start
    - `to`: optional ISO date for custom range end

    Implementation:
    1. Require admin authentication via `requireAdmin(event)`
    2. Fetch all completed analyses in date range with summary_json containing usage data
    3. Fetch pricing_tables to get input/output costs per model
    4. For each analysis, calculate AI cost:
      - Extract tokens: `summary_json._debug.usage.input_tokens` + `output_tokens`
      - Get model: `summary_json._debug.model_used` or infer from tier
      - Calculate: `(input_tokens / 1000 * input_cost) + (output_tokens / 1000 * output_cost)`
    5. Group by tier (Basic/Premium/Forensic) and calculate:
      - Total analyses
      - Total revenue (credits_used * credit_price, assume $0.99/credit for calculation)
      - Total AI cost (sum of individual analysis costs)
      - Gross margin = revenue - AI cost
      - Margin percentage = (margin / revenue) * 100
    6. Return format:
      ```json
      {
        "by_tier": [
          {
            "tier": "Basic",
            "analyses": 150,
            "revenue": 148.50,
            "ai_cost": 22.30,
            "gross_margin": 126.20,
            "margin_percent": 84.9,
            "avg_cost_per_analysis": 0.15
          },
          { "tier": "Premium", ... },
          { "tier": "Forensic", ... }
        ],
        "summary": {
          "total_revenue": 500.00,
          "total_ai_cost": 75.00,
          "total_margin": 425.00,
          "overall_margin_percent": 85.0,
          "blended_cost_per_analysis": 2.50
        },
        "period": { "from": "...", "to": "..." }
      }
      ```

    Handle missing usage data gracefully (skip analyses without _debug.usage).

<task type="auto">
  <name>Task 2: Create user update endpoint for credit adjustments and suspension</name>
  <files>server/api/admin/users/[id].patch.ts</files>
  <action>
    Create new endpoint `/api/admin/users/[id]` (PATCH) for user management operations.

    Request body:
    ```typescript
    {
      action: 'add_credits' | 'remove_credits' | 'suspend' | 'unsuspend',
      amount?: number,      // Required for credit actions
      reason: string        // Required for all actions (audit trail)
    }
    ```

    Implementation:
    1. Require admin authentication via `requireAdmin(event)`
    2. Validate request body with Zod:
      - `action`: enum of allowed actions
      - `amount`: positive integer, required for credit actions
      - `reason`: non-empty string, required for all
    3. Handle each action:
      - **add_credits**: `UPDATE users SET credits = credits + amount WHERE id = userId`
      - **remove_credits**: Same with subtraction, ensure credits don't go negative
      - **suspend**: `UPDATE users SET is_suspended = true, suspension_reason = reason, suspended_at = NOW()`
      - **unsuspend**: `UPDATE users SET is_suspended = false, suspension_reason = NULL, suspended_at = NULL`
    4. Create audit trail in `transactions` table:
      - For credit adjustments: `type='adjustment'`, `amount=0` (no money involved), `description=reason`
      - Log suspension changes in a separate audit log or user metadata
    5. Return updated user object

    Note: May need to add `is_suspended`, `suspension_reason`, `suspended_at` columns to users table via migration.
  </action>
  <verify>
    <automated>npm run test -- tests/unit/admin/user-management.test.ts --run</automated>
  </verify>
  <done>
    User update endpoint handles credit adjustments and suspension with audit trail logging
