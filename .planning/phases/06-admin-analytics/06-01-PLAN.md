<task type="auto">
  <name>Task 1: Create revenue API endpoint with time range aggregation</name>
  <files>server/api/admin/revenue.get.ts</files>
  <action>
    Create new endpoint `/api/admin/revenue` that returns revenue data aggregated by time period.

    Query parameters:
    - `range`: 'day' | 'week' | 'month' | 'quarter' | 'custom'
    - `from`: optional ISO date string for custom range start
    - `to`: optional ISO date string for custom range end

    Implementation:
    1. Require admin authentication via `requireAdmin(event)`
    2. Parse query params, calculate date range based on `range` param
    3. Query `transactions` table:
      - Filter: `status='completed' AND type IN ('purchase', 'refund', 'adjustment')`
      - Group by: DATE_TRUNC based on range (day/week/month/quarter)
      - Aggregate: SUM(amount) as revenue, COUNT(*) as transaction_count
      - Also sum credits_purchased for "credits sold" metric
    4. Return format:
      ```json
      {
        "revenue": [
          { "date": "2026-03-01", "gross_revenue": 49.90, "net_revenue": 44.91, "transactions": 10, "credits_sold": 100 }
        ],
        "summary": {
          "total_revenue": 499.00,
          "total_transactions": 50,
          "total_credits_sold": 500
        }
      }
      ```

    For package breakdown (5/10/25 credit packages), infer from amount:
    - ~$4.99 = 5 credits, ~$8.99 = 10 credits, ~$19.99 = 25 credits
    - Return as `by_package` array in response
  </action>
  <verify>
    <automated>npm run test -- tests/unit/admin/revenue.test.ts --run</automated>
  </verify>
  <done>
    Revenue endpoint created and tested, returns time-series data with gross/net revenue, transaction counts, and package breakdown
  </done>
</task>

<task type="auto">
  <name>Task 2: Create conversion funnel API endpoint</name>
  <files>server/api/admin/funnel.get.ts</files>
  <action>
    Create new endpoint `/api/admin/funnel` that returns conversion data for 4-stage funnel.

    Query parameters:
    - `range`: '7d' | '30d' | '90d' | 'custom'
    - `from`: optional ISO date for custom range start
    - `to`: optional ISO date for custom range end

    Funnel stages to reconstruct:
    1. **Signups**: Count of users created in range (from `users.created_at`)
    2. **Email verified**: Users with `email_confirmed_at` set (track when confirmed_at falls in range)
    3. **First analysis**: Users with at least one analysis (MIN(analyses.created_at) in range)
    4. **First purchase**: Users with at least one completed transaction (MIN(transactions.created_at) in range)

    Implementation:
    1. Require admin authentication
    2. Calculate date range from params
    3. Run 4 separate queries (or CTE) to count users at each stage
    4. Calculate conversion rates: stage_n / stage_(n-1) * 100
    5. Return format:
      ```json
      {
        "funnel": [
          { "stage": "Signups", "count": 100, "rate": 100 },
          { "stage": "Email Verified", "count": 85, "rate": 85 },
          { "stage": "First Analysis", "count": 60, "rate": 70.6 },
          { "stage": "First Purchase", "count": 25, "rate": 41.7 }
        ],
        "period": { "from": "2026-03-01", "to": "2026-03-15" }
      }
      ```
  </action>
  <verify>
    <automated>npm run test -- tests/unit/admin/funnel.test.ts --run</automated>
  </verify>
  <done>
    Funnel endpoint created and tested, returns 4-stage conversion data with counts and rates
  </done>
</task>
